# 现场 Mobile 布局与 UI 规范

本文档描述 **Sentri 现场端（Mobile）** 的固定信息架构、视觉语言与实现约定。**后续新增任何 Mobile 现场模块（生产、转舍、巡检、淘汰等）应统一放在 Mobile 模式下，并遵循同一套布局与 UI 形态**，仅在业务数据与表单字段上扩展。

---

## 1. 设计原则

- **选址优先**：首屏最上方完成 **车间 / 房间** 选址，再展示当前范围内的全部任务。
- **任务聚合首页**：首页列表为 **跨业务类型的任务卡片**（接种、生产、转舍等），用 **Tag 颜色 + 类型标签** 区分。
- **车间 → 房间分流**：当选址为 **车间** 时，需要落到具体栏位执行的任务（如接种）点击后通过 **底部 Drawer** 选择房间，再进入 **猪只/执行对象列表**；选址为 **房间** 时 **省略 Drawer**，直接进入列表。
- **猪只级列表**：列表卡片必须突出 **栏位号、猪只号（耳标）**，次要行为展示任务摘要（名称、计划时间、状态等）。
- **主流大厂移动端规范**：参考字节、阿里常见移动业务产品的密度与节奏；正文 14px 左右，辅助信息 12px，标题 18-22px，列表行距 1.45-1.65；触控区不小于 44px；顶部需保留安全区呼吸感；Segmented / Button 等控件文字必须视觉垂直居中；颜色可以大胆，但单页主色不超过 1 个、辅助强调色不超过 2 个。当前实现以深色侧栏、绿色主行动、白底次行动和低饱和浅底为基准；深色不大面积用于主按钮。
- **与 Console 同源**：数据与状态与 Console 共享；现场只做执行与闭环，不在此重复配置复杂规则。

---

## 2. 全局导航架构

Console 与 Mobile 在左侧导航顶部使用 `Radio.Group` 进行模式切换：

| 模式 | 菜单承载 | 页面承载 | 约束 |
|------|------|------|------|
| `Console` | Console 功能菜单，如免疫、猪只淘汰、设置 | 桌面管理页 | 不展示 Mobile 任务入口。 |
| `Mobile` | 不使用侧边功能菜单 | 线框屏幕预览 + 任务首页 | 所有 Mobile 功能通过首页任务卡进入，不按侧边栏功能管理。 |

Mobile 模式左侧只保留 Console/Mobile 模式切换，不承载业务功能导航；右侧主内容固定进入 Mobile 任务首页，用户通过首页任务卡进入接种、巡检、转舍等具体任务。

## 3. 信息架构（页面层级）

推荐屏幕状态机（可与接种模块一致或类比）：

| 层级 | 说明 |
|------|------|
| **Hub（任务首页）** | 顶部选址 + Tab（如「全部任务」「补打清单」）+ 任务卡片列表 |
| **PigList（对象列表）** | 某任务类型在 **当前房间** 下的明细列表（猪只行） |
| **Detail（详情）** | 单头猪/单条任务的完整信息与操作入口 |
| **Execute（表单）** | 执行记录、强校验后的提交 |
| **History（历史）** | 时间线式执行/异常日志 |

**返回栈**：详情需记录来源（`hub` | `pigList`），返回文案分别为「返回任务首页」「返回猪只列表」。

## 4. 首屏（Hub）布局结构（自上而下）

1. **状态栏 `.mv-statusbar`**
   - 展示时间与信号/Wi-Fi/电量，占位需接近最新版 iOS 细线状态栏风格，时间使用 SF / -apple-system 字体、17px 左右、600 字重；状态栏与首页头部保持紧凑间距，推荐状态栏约 44px、头部 padding-top 8-12px，图标不得被容器裁切；信号/Wi-Fi/电池应优先使用完整 SVG glyph 或等价完整图形，避免用拼接伪元素造成遮挡；作为 Mobile 首页的一部分，不代表具体机型外壳。

2. **顶部选择区 `.mv-topbar--reference`**
   - 左侧头像，中间使用组件库 `Select` 作为车间/单元选择器，右侧通知按钮。
   - 顶部不再展示“智慧猪场 · 现场任务”等副标题；下拉箭头不得使用手写字符，必须使用组件库默认图标。

3. **区域 Tab `.mv-section-tabs`**
   - 展示 `总览 / 单元1 / 单元2 / 单元3 / 单元4`。
   - 点击总览进入车间维度，点击单元进入房间维度。
   - 与概览数据区的下间距使用 24px。

4. **概览数据 `.mv-home-stats`**
   - 两列统计卡，展示存栏、巡检等关键指标；大数字约 24px / 700 字重，避免过大或过粗。
   - 与任务卡列表的下间距使用 20px。

5. **底部导航 `.mv-bottom-nav--reference`**
   - 首页底部操作栏高度约 64-66px。
   - 文案字号 12px 左右，图标 22px 左右，中间快捷按钮约 56px。
   - 不应挤占过多任务列表空间。

6. **任务卡片列表 `.mv-mission-list`**
   - 当前首页展示真实接种任务、断奶检查入口及保留的转舍演示任务；不再展示单元巡检、母猪体况评分演示卡。
   - 接种任务卡必须使用结构化信息布局：图标、固定标题“接种任务”、标题同行右侧下发状态、摘要、接种进度、进入提示分区展示；不再额外展示“接种任务”类型标签，不得通过隐藏核心摘要来解决重叠。
   - 卡片样式参考大圆角、轻阴影、左侧图标、右侧胶囊状态、进度条；任务卡纵向间距使用 16px；任务标题使用蓝灰/松石深色，不使用大面积纯黑。断奶检查首页入口需展示任务标题、生产线/批次、天数状态、已检查/需检查进度；不展示按钮，点击整张卡片打开选择房间底部抽屉。

7. **疫苗任务页 `.mv-piglist-root`**
   - 二级页面保留 Apple 风格状态栏，顶部使用返回 + 轻量主操作，标题不超过 19px。
   - 单元进度、目标列表、任务详情统一使用 22px 大圆角白色卡片、轻阴影、16px 模块间距。
   - 搜索、筛选、视图切换保持同一行；底部批量操作与结束任务按钮使用绿色主色，不使用大面积黑色按钮。
   - 断奶检查任务页需对齐现场任务页结构：顶部返回 + 居中标题、胶囊 Tab 两等分居中、Tab 下方固定展示单元进度告示板、猪只列表、搜索筛选工具栏、按栏位分组的列表卡；单元进度/猪只列表标题字号一致，进度文案避免过粗过大。任务详情页需用清晰的模块间距组织“总任务进度、关键指标、任务配置、SOP、状态说明”，避免字段挤在同一张无层级卡片中。检查登记弹窗使用底部 Sheet 风格，包含清晰头部、对象摘要、两张信息卡和固定提交操作。


## 4.1 截图复刻原则
- 截图只作为信息结构参考，用于识别模块顺序、字段、状态、交互入口和业务表达。
- 不按截图的实际像素间距、比例或不合理留白照抄；即使截图中元素间距很大，也必须按本项目 mobile 设计规范重新排布。
- 当截图与设计规范冲突时，设计规范优先，包括字号、行高、模块间距、卡片间距、按钮尺寸、Tab 等分、吸底操作、滚动区域和视觉重量。
- 复刻目标是还原业务结构与主流移动端体验，不是像素级临摹截图。
- 每次实现后必须自查：是否有元素挤压、错行、过粗、过大、视觉断层、按钮过重、滚动/吸底异常。

## 4.2 Mobile 字体重量约束
- Mobile 页面避免大面积使用 `760/800/900` 等重字重；常规标题建议 `650-700`，正文/说明建议 `400-600`。
- Mobile 页面顶部标题默认约 `17px / 650`，仅首页核心品牌/大标题可适度放大；任务执行、编辑、弹窗标题不允许使用又大又粗的视觉重量。
- Mobile 页面分标题必须低于页面标题层级，默认约 `14px / 620`；不得出现分标题比页面标题更大、更粗的层级倒挂。
- 耳标号、栏位号、任务数字等高频扫描信息不得又大又粗；默认使用约 `16px / 650`，仅关键汇总数字可适度放大。
- 大数字优先通过颜色和位置建立层级，不通过极粗字重制造强调。
- 每次 UI 调整必须检查是否存在过粗、过大、过黑的信息项，尤其是列表主字段、进度数字、弹窗头部。


## 4.3 Mobile 间距约束
- 模块标题到紧随其后的卡片/工具栏默认 `12px`，避免出现标题和内容断层。
- 同页面相同层级标题间距必须一致；不得因为截图留白大而放大标题与卡片距离。
- 同组卡片或列表项默认 `12-16px`；跨模块间距默认 `20-24px`。
- 表单/详情页中，上一张内容卡片结束到下一个模块标题默认 `20-24px`，例如仔猪信息页中“记录当前断奶数量”卡片到“仔猪体重”标题，不允许贴得过近。
- 每次 UI 调整必须检查：标题是否离卡片过远、卡片是否贴住下一个标题、列表工具栏是否与标题断层、卡片组间距是否一致。


## 4.4 Mobile 滚动约束
- 任务执行页优先使用页面级单一纵向滚动容器，避免“外层 `overflow:hidden` + 内层局部滚动”导致顶部卡片、Tab、进度板等区域滑不动。
- 页面内需要吸底操作栏时，优先使用 `position: sticky; bottom: 0`，不要额外制造第二个可滚动容器。
- 弹窗/抽屉可以独立滚动，但必须限制在弹层内部；主页面和弹层滚动不能互相抢事件。
- Mobile 抽屉必须展示在 mobile 预览屏内部，不允许挂载到 `document.body` 形成全局页面抽屉；抽屉打开后由屏内遮罩覆盖底层页面，抽屉内容区独立滚动，避免滑动穿透到底层猪只列表。
- 抽屉内容必须使用不透明背景和足够层级，吸底后不得在抽屉底部或圆角下方露出底层列表。
- 操作弹窗/操作抽屉打开时必须锁住底层任务列表滚动；此时上下滑动只能作用于当前弹窗/抽屉内容，不允许出现右侧外部列表滚动条或滚动穿透。
- 从长列表中打开的吸底操作弹窗必须挂载到 mobile 屏幕容器，而不是挂在当前滚动列表内容内；无论底层列表滚动到什么位置，弹窗都必须固定吸附在屏幕底部，关闭按钮必须始终可见可点。
- 若操作弹窗内还需要二级选择弹窗（如选择留种仔猪），二级弹窗也必须挂载到 mobile 屏幕容器并作为屏内吸底弹窗展示，不允许在当前表单页面内形成双层滚动抽屉。
- 每次新增任务执行页必须检查：从标题、Tab、进度卡、列表任意区域开始上滑/下滑，都应能正常滚动。

## 5. 车间 → 房间：底部 Drawer
- 所有 Mobile 底部抽屉必须吸附在线框屏幕底部，使用 `placement="bottom"` + 屏幕内绝对定位，不允许随内容区浮动。
- 检查任务入口点击后先进入选择房间底部抽屉，再进入对应检查执行页。

- 使用 **Ant Design `Drawer` `placement="bottom"`**，标题 **「选择进入房间」**。  
- 内容：每个房间一行 **大按钮**，文案包含 **待处理数 / 总头数**。  
- 样式容器：**`.mv-room-drawer`**（底部留白充足）。  
- 选中房间后：关闭 Drawer，进入 **PigList**，并带上 `batchId` / `roomId`（或新业务等价维度）。

---

## 6. 猪只列表（PigList）

- 顶栏：**返回任务首页**（清空列表上下文）。  
- 标题区：说明 **房间名** + **列表语义**（如「批次内本间全部头只」）。  
- **主操作**：**本间完成** 类按钮全宽、`large`（与接种逻辑一致时可复用）。  
- **筛选**：第二行 **`.mv-chip-row`**（全部 / 待执行 / 执行中 / 已完成 / 异常等）。  
- **列表项 `.mv-task-card` + `.mv-pig-row-card`**：  
  - **第一行**：左侧 **`.mv-pig-row-ids`** — **`.mv-stall-pill` 栏位** + **`.mv-ear-pill` 猪只号**；右侧 **状态 Tag**。  
  - **第二行**：任务主名称（如计划名）。  
  - **第三行**：`.mv-task-meta`（疫苗、单位、计划时间等，用 `·` 分隔）。  
  - 豁免等：**`.mv-hint` + Badge**。  
- **补打（Is_Retry）**：**`.mv-task-card--retry`** + 顶部 **`.mv-retry-banner`** 文案 **「补打 · Is_Retry」**；详情页顶部 **`.mv-retry-stripe`** 再次强调。

---

## 7. 详情（Detail）

- 顶栏：**`.mv-nav`** 内 `Button type="link"` 返回。  
- 豁免弱提示：**`Alert type="warning"` + `.mv-exempt-banner`**（黄底、圆角）。  
- 主信息卡：**`.mv-card.mv-hero`**  
  - 标题行：**栏位 + 猪只号**（`.mv-pig-id-lg`），右侧状态 Tag。  
  - **`.mv-kv-grid` 两列**：车间、房间、计划、疫苗、剂量、目标群、覆盖数、豁免命中、计划时间等（按业务删减）。  
- 异常面板：**`.mv-exception`**（浅橙渐变底）。  
- 底部操作：**`.mv-actions`** 纵向大按钮组（开始执行、正常完成、强制完成、查看历史等）。

---

## 8. 表单与历史

- 表单容器：**`.mv-form`**；表单项纵向，`DatePicker`/`InputNumber` 全宽。  
- 强制类操作：标题旁 **橙色 Tag** 标明「强制接种」等业务语义。  
- 历史：**`.mv-timeline` + `.mv-log-card`**，类型小标签 + 摘要 + 人/时间。

---

## 9. 全局容器与宿主

- 根节点：**`.mv-root`**（`max-width: 420px`，水平居中，底部 padding）。
- Mobile 模式内容区包裹：**`.mobile-vacc-host`**，负责居中展示 Mobile 主屏幕；Mobile 模式下内容区不保留 Console 白色页边。
- 主屏幕外框：**`.mobile-vacc-phone`** 只允许表现为线条屏幕框，不展示具体手机机型、状态栏、刘海或 Home Indicator。
- 弹层：Modal / Drawer 必须挂载在 **`.mobile-vacc-phone__screen`** 内，避免遮罩跑到桌面全屏。
- 全局字体与色板见 `src/styles.css` 顶部 `:root` 与 `body`。
- Mobile 单页配色应保持克制：一个主色承担品牌/主行动，少量辅助色承担风险或状态，不使用过多高饱和颜色。接种页当前使用绿色作为唯一主色，深色仅用于导航/文字层级，不大面积用于按钮；白底细边用于次行动，橙色仅用于风险提示。

## 10. CSS 类名索引（`src/styles.css`）

现场模块样式均以 **`mv-` 前缀** 命名，避免与 Console 混用。主要类名：

`mv-statusbar` `mv-section-selector` `mv-section-tabs` `mv-home-stats` `mv-root` `mv-header` `mv-title` `mv-sub` `mv-chip-row` `mv-chip` `mv-chip--on` `mv-list` `mv-task-card` `mv-task-card--retry` `mv-retry-banner` `mv-retry-stripe` `mv-scope-panel` `mv-scope-select` `mv-hub-context` `mv-hub-section` `mv-section-title` `mv-home-task-list` `mv-home-task-card` `mv-home-task-card--vaccine` `mv-home-task-title` `mv-home-task-sub` `mv-home-task-hint` `mv-pig-row-card` `mv-pig-row-main` `mv-pig-row-ids` `mv-stall-pill` `mv-ear-pill` `mv-room-drawer` `mv-card` `mv-empty` `mv-nav` `mv-hero` `mv-kv-grid` `mv-exempt-banner` `mv-exception` `mv-actions` `mv-form` `mv-timeline` `mv-log-card`

新增模块时 **优先复用上述类**；确需新样式时继续 **`mv-` 前缀** 并补记到本文档「类名索引」。

## 12. 新模块落地检查清单

- [ ] 首屏是否包含 **车间/房间 Segmented + Select**，且筛选逻辑清晰。  
- [ ] 首页任务卡片是否 **标题 + 右侧下发状态 + 副标题** 与现有一致。  
- [ ] 车间维度是否需要 **底部 Drawer 选房间**；房间维度是否 **直达列表**。  
- [ ] 列表行是否包含 **栏位号 + 猪只号** pill，且语义与 `pigMeta` 一致。  
- [ ] 补打/特殊任务是否有 **retry 横幅/条** 与接种一致。  
- [ ] 是否放在 **Mobile 模式任务首页** 的任务卡入口，不使用侧边栏管理业务功能。  
- [ ] 是否使用 **`.mobile-vacc-phone` 线框屏幕**，没有展示具体机型外观。  
- [ ] 是否使用 **`.mv-root` + `.mv-*`**，未破坏 Console 桌面布局。  
- [ ] 详情返回是否处理 **多层级返回栈**。
