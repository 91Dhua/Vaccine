# UI 变更自检清单

用于约束后续前端样式与布局修改，避免“只改了一个点，却把附近布局带坏”的问题。

## 1. 局部改动联动检查
- 修改任一输入框、按钮、标题、Tag、表格列宽后，必须回看同模块整块布局，不只看当前元素。
- 必须检查当前改动是否影响：同一行相邻元素、同一卡片右侧按钮、上下说明文案、响应式断点下的换行。
- 如果一个模块有桌面态和窄屏态，两个状态都要检查，不能只验证当前窗口尺寸。

## 2. 对齐检查
- 输入框与单位必须默认保持同一行，除非是明确设计为多行表单。
- 卡片标题区中的说明块与主操作按钮，默认优先保持同一行；只有在窄屏断点下才允许折行。
- 当某个模块与相邻模块采用同一信息架构时，标题层级、控件高度、按钮位置应保持一致。

## 3. 变更后核验
- 改动完成后，至少执行一次构建校验。
- 改动样式时，需搜索是否存在更高优先级或更晚声明的样式覆盖。
- 若发现布局由媒体查询或通用样式导致异常，应修正断点或局部选择器，而不是只改当前元素尺寸。

## 4. 状态反馈检查
- 用户完成选择、勾选、添加后，页面主入口按钮应优先给出即时反馈，例如颜色变化、数量文案（如 `已选 2 头`）。
- 对用户已经完成的关键操作，不要只把结果藏在弹窗里；关闭弹窗后，外层页面要能一眼看出“已经选了什么、选了多少”。
