## 1. 日志类型与界面显示

本节仅覆盖日志类型与 UI 展示；全文结构见主文档。

## 2. 核心日志类型

当前已记录的日志类型包括：

1. `created`（创建/入库）
2. `lab_tested`（实验室检测）
3. `diluted`（稀释）
4. `reserved`（预留）
5. `picked`（拣选）
6. `used`（使用）
7. `moved`（移动）
8. `adjusted`（调整）
9. `discarded`（废弃）
10. `manual_entry`（手动录入）
11. `expired`（过期）

## 4. Details 字段格式规则

`Details` 列应当：

1. 使用人类可读文本，而非原始 JSON；
2. 包含能够解释变更的关键字段；
3. 优先简洁短语，而非完整长句；
4. 在相关情况下包含母猪 ID、预留 ID、拣货单 ID、公猪 ID 等标识。

## 5. 手动登记在界面中的显示

对非 Sentri 精液编码登记流程，除非未来引入专门事件类型，否则日志行应显示为 `manual_entry`。

`Details` 字段至少应包含：

1. 扫描码上下文；
2. 公猪 ID（boar ID）；
3. 品种（breed）；
4. 来源场（source farm）；
5. 到期日（expiration date）。
