# 免疫全链路数据流转盘点（Console ↔ 猪舍 ↔ Console）

本文档与产品描述 **逐段对齐**，并标注当前 **Vaccine Codex 仓库中的落地情况**（✅ 已有 / 🟡 部分 / ⬜ 未做），便于后续迭代同一套数据模型与接口。

---

## 阶段一：计划制定与下发（Console）

| 模块 | 业务要点 | 实现状态 |
|------|----------|----------|
| **基础配置** | 计划名称、疫苗与剂量、时间框架（普免或阶段/事件触发） | 🟡 计划与任务流在 `VaccinePlanPage`、`VaccineTaskWizard`、`App` 的 `tasks` 中；普免/跟批粒度需与「计划实体 ID」长期绑定 |
| **目标圈选 (Targeting)** | 高级设置规则引擎：品种、胎次、来源、体重区间等静态/结构性特征 | 🟡 计划侧有目标猪群与复杂配置 UI；任务侧有 `pigIds`、`targetPigGroupLabel`；**规则引擎输出与每次下发名单的可追溯 ID** 建议显式存储 |
| **安全防线 (Exemption)** | 动态状态不可打：妊娠后期、患病、用药中等 | 🟡 Mobile 有 `exemptionHit` / `exceptionPending` 与异常处理；**与计划里豁免规则配置需同源下发**（同一规则版本号更佳） |
| **效果追踪配置** | 采样开关、抽检间隔（如 28 天）、抽样比例、合格率阈值（如 80%） | 🟡 **Console 计划配置已支持**（普免 / 跟批表单 + 列表列 + 保存 `effectTracking`）；**未**据此自动触发阶段三采样任务（待调度与 Mobile 任务类型） |

**阶段一向阶段二的连接**：计划到点（或事件）→ 生成 **接种任务**（含圈选结果、豁免标签、可选的效果追踪参数）→ 进入 Mobile 可执行队列。

---

## 阶段二：现场执行与动态决策（Mobile）

| 模块 | 业务要点 | 实现状态 |
|------|----------|----------|
| **任务接收** | 按栋舍 → 房间 → 栏位聚合展示接种任务 | ✅ `MobileVaccinationPage` + `mobileWorkshops` + `pigMeta`（车间/房间/栏位/耳标） |
| **柔性执行 (Double Check)** | 正常：扫码/勾选完成；豁免/异常：弱提示 → **强制接种**（需原因）或 **标记暂缓** | 🟡 有弱提示 Banner、强制接种确认、暂缓并生成补打（`Is_Retry`）；**强制接种「填原因」**可加强为必填表单项 |
| **实时补打池** | 暂缓猪只脱离主任务，进入 **今日待办 · 补打清单** | ✅ 补打清单 Tab、`spawnRetryTask`、`isRetry` 视觉标识 |
| **房间结案确认** | 「本间完成」时扫描房间状态，提醒遗留暂缓猪只，确认后转入补打/待办 | ✅ `confirmRoomComplete` + 弹窗文案与生成补打逻辑 |

**阶段二向阶段三的连接**：若计划开启效果追踪且到达 **抽检间隔**，应由调度生成 **采样任务**（新任务类型），携带 **关联计划 ID / 原接种批次或任务 ID / 目标抽样策略**。

---

## 阶段三：效果抽检与化验（Mobile + Console）

| 模块 | 业务要点 | 实现状态 |
|------|----------|----------|
| **采样触发** | 达间隔后自动向 Mobile 下发 **抽血采样任务**，并随机指定关联猪只名单 | ⬜ 无采样任务类型、无调度、无名单存储 |
| **灵活采样** | 指定猪只淘汰/死亡时，现场 **更换样本猪** | ⬜ 无 |
| **结果录入** | 实验室结果在 Console **免疫效果审核** 页录入：实际抽样数、阳性（合格）数等 | ⬜ 页面与 API 未实现（曾用临时抽屉已按产品要求回退） |

**阶段三向阶段四的连接**：每条「待审核化验批次」应唯一关联 **计划/任务/采样批次**，并携带 **阈值、目标抽样数、实录样本数、阳性数** 供实时计算合格率。

---

## 阶段四：智能决策与闭环（Console）

| 分支 | 业务要点 | 实现状态 |
|------|----------|----------|
| **PASS** | 合格率 ≥ 阈值 → 免疫成功，**原始计划归档闭环** | ⬜ 需批次状态机 + 计划归档标志 + 与未完成任务校验 |
| **FAIL** | 合格率 < 阈值 → 红色预警 | ⬜ |
| **兽医复核** | 必须勾选确认「排除野毒感染风险」后才可生成补免 | ⬜ |
| **自动生成补免** | 按 **最新实时存栏**（剔除死淘）克隆 **原疫苗与剂量**，生成带 **【效果补免】** 标签的新任务，回到阶段二 | ⬜ 需在 `TaskRow`（或等价结构）上增加任务类型/标签、`sourceTaskId`、存栏快照策略 |

---

## 建议的全局主键与事件（便于后端与前端对齐）

1. **计划 ID**（普免/跟批）— 阶段一创建，贯穿归档。  
2. **接种任务 ID** — 每次下发执行（阶段一→二）。  
3. **采样任务 ID** — 阶段三独立任务，**关联**接种任务或计划 + 采样批次号。  
4. **化验批次 ID** — 阶段三结束、阶段四审核的载体（录入阳性/样本数）。  
5. **补免任务 ID** — 阶段四 FAIL 生成，`kind = 效果补免`，`parent` 指向原接种或化验批次。

---

## 小结

- **已贯通**：Console 任务 → Mobile 按车间/房间/栏位执行 → 豁免与双检 → 暂缓/补打/本间结案。  
- **待贯通**：**效果追踪参数**（已写入计划）→ **采样任务**（Mobile）→ Console **免疫效果审核** → **PASS/FAIL 状态机** → **效果补免任务** 再次进入 Mobile。

后续开发建议顺序：**扩展计划与任务模型（含效果追踪）→ 采样任务（Mobile）→ 审核列表 + 表单（Console）→ 结案与补免写回任务**。
