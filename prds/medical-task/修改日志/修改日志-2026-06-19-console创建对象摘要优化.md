# 修改日志：Console 创建对象摘要优化

## 变更日期

2026-06-19

## 影响文件

- `src/components/MedicalTaskSchemePage.tsx`
- `src/styles.css`
- `prds/medical-task/00-疫苗任务与治疗任务统一入口方案.md`

## 变更原因

创建治疗任务时，顶部创建对象摘要原先会展示选中猪只耳标号。如果一次选中很多猪，只展示前几个耳标既不完整，也会让用户误以为只有这些猪进入任务；展示更多耳标又会造成页面拥挤。因此该摘要应只表达创建方式、目标数量和来源，不展示耳标号。

## 用户可见变化

- 任务配置页顶部创建对象摘要不再展示猪只耳标号。
- 摘要改为三块信息：创建对象、目标猪只、来源。
- 手动勾选场景展示来源为 `健康诊疗 · 手动勾选`。
- 按疾病批量创建场景展示疾病/症状标签。
- 侧边治疗对象摘要同步移除 `猪只示例`。
- 摘要 banner 改为固定网格布局，减少信息拥挤和视觉散乱。

## PRD / Mock Data 更新

- PRD 增加顶部创建对象摘要不展示耳标号的规则。
- Mock 数据无结构变化。

## 验证执行

- `npm run build`
- 浏览器验证 `http://127.0.0.1:5173/` Console 治疗「先选任务类型」：
  - 手动勾选猪只进入任务配置页后，顶部摘要不展示耳标号。
  - 摘要展示创建对象、目标猪只数量和来源。
  - 页面布局不因选中猪只数量变化而扩展耳标文本。
