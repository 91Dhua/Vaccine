# 修改日志：Console 治疗筛选与配置布局

## 变更原因

实际生产场景下猪只规模可能达到十万级，Console 治疗创建页不应初始展示全量猪只列表，也不应让用户从全量列表中逐条查找。任务配置页也需要保持疫苗、兽药、保健品之间的页面结构一致，降低用户进入不同任务类型后的陌生感。

## 用户可见变化

- 健康诊疗页初始不展示猪只列表，只展示筛选区和空状态。
- 用户必须先使用栏位/舍、疾病标签、症状标签或健康诊疗状态筛选后，才展示匹配猪只。
- 猪只 ID 通过搜索实现，只在已有筛选结果内继续缩小范围；仅输入 ID 不展示猪只。
- 筛选条移除解释文案，重置按钮放在筛选控件旁边。
- 筛选标题与输入框同高并对齐，避免筛选区视觉错乱。
- 筛选输入框、下拉框占位文案、已选标签和关闭图标垂直居中展示。
- 疫苗、兽药、保健品任务配置页使用一致的任务配置页面骨架。

## PRD / Mock Data 更新

- PRD 增加 Console 选猪筛选规则。
- PRD 增加任务配置页面一致性规则。
- Mock 猪只默认不再预选，避免初始未筛选时仍存在隐性选中对象。
