# 修改日志：Console 任务确认预览布局

## 变更日期

2026-06-29

## 影响文件

- `src/components/MedicalTaskSchemePage.tsx`
- `src/styles.css`
- `prds/medical-task/00-疫苗任务与治疗任务统一入口方案.md`

## 变更原因

治疗「先选任务类型」流程进入任务确认预览页后，页面同时展示顶部创建对象摘要、左侧治疗对象卡片和右侧确认信息卡片，信息重复且左右留白比例不均，用户需要在多个区域之间来回确认任务内容。

## 用户可见变化

- 确认下发页改为单一确认面板，不再额外展示左侧治疗对象卡片。
- 面板顶部集中展示任务类型、目标猪只数量和任务标签。
- 面板主体按 `任务信息` 与 `执行对象` 分组展示接种/治疗内容、执行时间、目标猪只、创建方式、来源和疾病/症状标签。
- 底部提示与 `上一步`、`确认下发` 操作放在同一确认区，减少操作区和预览信息的割裂感。

## PRD / Mock Data 更新

- PRD 补充确认下发页使用单一确认面板、按任务信息和执行对象分组展示的规则。
- Mock 数据无结构变化。

## 验证执行

- `npm run build`
- 浏览器验证 `http://127.0.0.1:5173/` Console 治疗「先选任务类型」：
  - 按疾病批量创建进入确认页后，确认信息展示为单一面板。
  - 面板内任务信息与执行对象分组清晰，未出现文字重叠或横向溢出。
  - 底部提示、`上一步` 和 `确认下发` 位于同一确认区内。
