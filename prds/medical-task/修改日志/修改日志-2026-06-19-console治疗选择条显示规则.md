# 修改日志：Console 治疗选择条显示规则

## 变更日期

2026-06-19

## 影响文件

- `src/components/MedicalTaskSchemePage.tsx`
- `prds/medical-task/00-疫苗任务与治疗任务统一入口方案.md`

## 变更原因

筛选出猪只后，如果用户尚未勾选任何猪只，页面不应展示绿色批量选择条和创建任务按钮。该区域属于“已选对象后的批量操作反馈”，提前展示会让用户误以为已经产生了待创建对象。

## 用户可见变化

- 使用筛选条件展示猪只后，未勾选猪只时不展示绿色选择条。
- 用户勾选至少一头猪后，才展示绿色选择条、已选数量、匹配数量和创建任务按钮。
- 取消全部勾选后，绿色选择条再次隐藏。

## PRD / Mock Data 更新

- PRD 增加绿色批量选择条仅在已勾选猪只后展示的规则。
- Mock 数据无字段变化。

## 验证执行

- `npm run build`
- 浏览器验证 `http://127.0.0.1:5173/` Console 治疗「先选任务类型」：
  - 选择疾病标签后展示匹配猪只，但不展示绿色选择条。
  - 勾选一头猪后展示绿色选择条和创建任务按钮。
  - 取消勾选后绿色选择条隐藏。
