# 修改日志 - 2026-07-03 - Mobile库存入库盘点统一

## 变更日期

2026-07-03

## 受影响文件

- `Vaccine/src/components/inventory/MobileInventoryToolsPage.tsx`
- `Vaccine/src/styles.css`
- `Vaccine/src/components/inventory/inventoryUtils.test.ts`
- `Inventory management/Inventory PRD/02-Mobile库存管理PRD.md`

## 变更原因

Mobile 库存工具页入口过多且展示占屏较大；入库、盘点流程与 Console 批量入库和盘点闭环不一致，容易让现场端形成另一套库存操作口径。

## 用户可见变化

- Mobile 库存首页只保留 `查库存`、`入库`、`盘点` 三个入口，并改为紧凑方块展示。
- `新增物料` 不再作为工具箱独立入口，仅在入库条目内搜索不到物料时内联展开。
- Mobile 入库改为按 Console 批量入库口径执行：选择分类、添加多个入库条目、选择或内联新建物料、填写数量/单位/到期日/总进货价/供应商/存放位置后统一确认入库。
- Mobile 盘点改为按 Console 盘点口径执行：选择盘点方式、确认范围、录入实盘总数、差异确认后提交；提交后系统自动处理容差内小差异，剩余差异进入 `盘点差异处理`。

## PRD / Mock 数据更新

- 更新 Mobile 库存 PRD：入口改为三个功能方块；入库内快速新增物料；盘点改为盘点方式、范围确认、实盘录入、差异确认闭环。
- 本次未新增静态 mock 数据；Mobile 页面复用现有库存物料、批次、流水 mock 数据，并在页面状态内生成入库批次、采购入库流水和盘点差异结果。

## 验证

- 已执行 `npx tsx src/components/inventory/inventoryUtils.test.ts`。
- 已执行 `npm run build`。
- 新增回归断言：Mobile 库存入口不得恢复独立新增物料；Mobile 入库必须复用批量入库条目、快建物料、包装换算和提交校验口径；Mobile 盘点必须复用范围选择、差异生成与自动处理规则。
