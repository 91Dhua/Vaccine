# 修改日志 - 库存首页采购单按钮按需显示

- **日期**：2026-07-08
- **影响文件**：
  - `src/components/inventory/ConsoleInventoryPage.tsx`
  - `src/components/inventory/inventoryUtils.test.ts`

## 变更原因

库存管理首页的主要目的是查看库存状态与风险，「生成采购单」在未选中物料时持续展示会分散注意力，且禁用态按钮仍可能误导用户以为这是页面主操作。

## 用户可见变化

- 物料列表未勾选任何行时，不再展示「已选 0 种」与「生成采购单」按钮。
- 勾选至少 1 种物料后，右侧才出现已选数量与「生成采购单」入口。

## 验证

- `npm run build`
- `npx tsx src/components/inventory/inventoryUtils.test.ts`
