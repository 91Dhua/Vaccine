# 修改日志-2026-06-26-库存首页数据卡片样式修复

## 变更日期

2026-06-26

## 受影响文件

- `src/styles.css`
- `Inventory management/Inventory PRD/01-Console库存管理PRD.md`
- `Vaccine/prds/inventory/修改日志/修改日志-2026-06-26-库存首页数据卡片样式修复.md`

## 变更原因

库存首页经营分析区内 `最大占比`、`本月累计`、`日均消耗`、`峰值日`、`消耗天数` 等数据展示使用了外层胶囊容器，同时内部 `Text` 标签又继承了同一套包边样式，形成胶囊嵌套胶囊，视觉噪音偏高，不符合 Sentri 轻量后台操作风格。

## 用户可见变化

- 库存首页上方 `库存金额占比` 卡片右侧的 `最大占比` 改为紧凑的标签 + 数值块。
- `库存消耗趋势` 图表上方的累计、日均、峰值日、消耗天数指标改为统一的小型数据块。
- 指标内部不再出现二次包边的嵌套胶囊，整体边框半径、间距和文字层级更克制。

## PRD / Mock 数据更新

- Console 库存管理 PRD 已补充首页经营分析辅助指标的轻量展示规则。
- 本次仅修复展示样式，不涉及业务字段、计算逻辑或 mock 数据更新。

## 验证

- `npm run build`：通过。
- `npx tsx src/components/inventory/inventoryUtils.test.ts`：通过（退出码 0）。
- 已在本地浏览器打开 `http://127.0.0.1:5173/` 并进入 `库存管理`：确认 `最大占比`、`本月累计`、`日均消耗`、`峰值日`、`消耗天数` 外层为 8px 轻量数据块，内部标签不再带边框或胶囊背景。
