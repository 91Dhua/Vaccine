# 修改日志：Mobile 入库滚动与必填标识

## 变更日期

2026-07-03

## 影响文件

- `Vaccine/src/components/inventory/MobileInventoryToolsPage.tsx`
- `Vaccine/src/styles.css`
- `Inventory management/Inventory PRD/02-Mobile库存管理PRD.md`

## 变更原因

Mobile 入库页面底部固定操作区和主导航会遮挡表单末尾，导致用户无法稳定滚动到最后的选填项和备注。同时移动端入库字段未显性展示必填星号，用户不容易判断哪些字段必须填写；不同分类的新物料必填项需要与 Console 批量入库保持同一口径。

## 用户可见变化

- Mobile 入库页面底部增加安全滚动空间，展开选填项后可以继续滚动到备注和页面最底部。
- 底部 `清空 / 确认入库` 操作区调整为紧凑固定工具条，贴在底部主导航上方，不再占用大面积页面空间或与主导航重叠。
- `添加入库物料` 保持在表单内容流中，滚动到底部时不会被固定操作区或底部导航遮挡。
- 入库基础必填字段增加红色星号：物料名称、入库数量、单位、到期日期、总进货价、供应商、存放位置。
- 新建物料时，规格与包装展示为必填。
- 快速新增物料中的分类专属字段继续复用 Console 批量入库字段定义，并在 Mobile 端显示 Ant Design 必填星号。

## PRD / Mock 数据更新

- `02-Mobile库存管理PRD.md` 补充 Mobile 入库必填星号规则、各分类专属必填字段需复用 Console 批量入库口径、底部固定操作区不得遮挡最后字段。
- Mock 数据未新增；本次调整复用现有库存物料、分类字段和移动端入库演示数据。
