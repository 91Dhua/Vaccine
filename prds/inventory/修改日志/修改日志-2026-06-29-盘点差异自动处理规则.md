# 修改日志 - 2026-06-29 - 盘点差异自动处理规则

## 变更日期

2026-06-29

## 修改原因

盘点差异当前需库管逐条人工选择既有处理方式，日常小差异、期间库存变动、任务多扣、漏记出库等场景占用较多人工判断时间。本次按 Oracle WMS、SAP EWM 等主流做法，对盘点后处理 PRD 进行修正：系统先剔除假差异，再自动处理小差异，再匹配近期业务流水，仍无法判断时触发盲盘重盘，最后兜底调账并在大额场景标记财务待确认。

## 受影响文件

- `prds/inventory/02-盘点差异自动处理规则.md`（修正盘点后处理自动规则）

## 用户可见变化（PRD 定义，待开发）

- 提交盘点后系统自动处理大部分差异并更新库存，不再默认要求进入差异列表逐条点选。
- 盘点期间发生出入库时，系统按「盘点开始快照 + 盘点期间已确认出入库」重新计算有效账面，避免生成假差异。
- 差异状态口径修正为：`待自动处理`、`待重盘`、`待人工确认`、`已处理`、`已取消`；处理来源区分 `自动` / `人工`。
- 「盘点差异处理」页改为异常差异队列，角标仅统计待重盘 + 待人工确认。
- 新增盲盘重盘页；超容差且无法匹配时最多自动重盘 2 次。
- 重盘仍无法解释时，系统兜底按 `原因不明盘盈/盘亏` 调账；大额或成本缺失场景标记 `财务待确认`，但不阻塞库存更新。
- 首页 / 物料详情 `含未处理差异` 仅在仍有待重盘或待人工确认时展示。

## 规则摘要

- Step 0：用快照 + 期间流水校正有效账面，校正差异为 0 时不进入异常队列。
- Layer 1：容差内自动 `原因不明盘盈/盘亏`，金额只决定是否财务待确认，不阻塞库存更新。
- Layer 2：近 14 天业务流水唯一匹配时自动精准处理，覆盖入库更正、任务多扣、漏记任务出库、入库多录、过期报废等。
- Layer 3：无候选时生成盲盘重盘任务；多个候选时进入人工确认，不自动猜。
- Layer 4：重盘 2 次后仍无法解释时兜底 `原因不明盘盈/盘亏`；大额或成本缺失记财务待确认。
