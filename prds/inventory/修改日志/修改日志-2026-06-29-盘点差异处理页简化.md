# 修改日志 - 2026-06-29 - 盘点差异处理页简化

## 变更日期

2026-06-29

## 修改原因

盘点差异处理页当前只需要承载待处理差异。顶部 `待处理 / 已处理 / 财务待确认` 统计卡重复占用空间；`处理状态` 列在本页恒为待处理；财务模块尚未实现，`财务状态` 暂不应作为主列表信息展示。处理完成后会生成库存流水，用户可在库存流水中追溯结果。

## 受影响文件

- `src/components/inventory/ConsoleInventoryPage.tsx`
- `prds/inventory/02-盘点差异自动处理规则.md`

## 用户可见变化

- 移除盘点差异处理页顶部三块统计展示。
- 移除列表中的 `处理状态` 与 `财务状态` 两列。
- 列表只展示待处理盘点差异；处理完成后该记录从列表消失。
- 处理完成提示改为 `差异已处理，库存流水已生成`。
- 空态文案改为 `暂无待处理盘点差异`。

## PRD / Mock 数据更新

- PRD 明确该页面不展示独立统计卡，不展示处理状态与财务状态。
- 本次未新增 mock 数据，沿用现有待处理差异数据验证列表简化。

## 验证

- 已运行 `npm run build`，类型检查与生产构建通过。
