# 修改日志 - 2026-07-08 - 库存安全库存移除与入库草稿提示

2026-07-08

## 变更原因

当前产品没有安全库存设置入口，继续保留安全库存和低库存风险会造成业务口径不闭环。入库页恢复草稿时也需要给用户明确但轻量的状态反馈。Ant Design 的 `destroyOnClose` 已弃用，需要在本次相关库存/物料弹窗中改为 `destroyOnHidden`。

## 用户可见变化

- 库存风险不再生成或展示低库存风险，首页风险筛选调整为已过期、临期库存、负库存。
- 物料详情、物料编辑、物料管理和入库新建物料流程均不再展示或保存安全库存。
- 入库预览页单据头改为只读展示，用户需要点击返回修改后才能编辑供应商、入库日期、采购单号和备注。
- 入库页恢复本地未提交草稿时，在页面顶部展示 `已恢复上次未提交草稿` 的轻量提示。
- 本次相关库存/物料弹窗和抽屉改为使用 `destroyOnHidden`。

## PRD / Mock 数据更新

- 移除 `低库存提醒` mock 数据。
- Console / Mobile 库存 PRD 同步移除安全库存和低库存风险口径。
