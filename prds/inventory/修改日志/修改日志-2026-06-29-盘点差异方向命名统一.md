# 修改日志 - 2026-06-29 - 盘点差异方向命名统一

## 变更日期

2026-06-29

## 修改原因

盘点差异列表中的 `差异方向` 实际表达的是盘点业务里的 `盘盈 / 盘亏`。继续展示 `实盘多出 / 实盘少了` 会与库存流水、盘盈调整批次、盘亏扣减等既有业务概念不一致，因此统一命名。

## 受影响文件

- `src/components/inventory/inventoryData.ts`
- `src/components/inventory/ConsoleInventoryPage.tsx`
- `prds/inventory/02-盘点差异自动处理规则.md`

## 用户可见变化

- 盘点差异确认页、盘点完成页、盘点详情页、盘点差异处理页中的方向文案统一为 `盘盈` / `盘亏`。
- 盘点差异处理列表列名由 `差异方向` 调整为 `盘点结果`。
- 物料详情未处理盘点差异提示改为 `盘盈 +数量` 或 `盘亏 数量`。

## PRD / Mock 数据更新

- PRD 中的差异方向说明同步改为 `盘点结果`、`盘盈`、`盘亏`。
- Mock 差异数据的方向枚举同步改为 `盘盈` / `盘亏`，处理方式匹配仍沿用现有盘盈、盘亏相关处理方式。

## 验证

- 已运行 `npm run build`，类型检查与生产构建通过。
